## 事前準備 (発表 30 分前まで)

### 環境チェック

- [ ] サーバ起動: `.\start.ps1`
- [ ] ブラウザで `http://127.0.0.1:8787` を開く
- [ ] **キャッシュをクリア** (Ctrl+Shift+R)
- [ ] **スキャンタブ** が初期表示になっていることを確認
- [ ] フッタ ステータスバーに **緑のドット** が表示されている (接続中)
- [ ] **過去のスキャン結果が DB に残っている** ことを確認 (結果タブ・ホストタブで内容あり)
- [ ] 「**このパソコンを検査**」カードに **「✓ 管理者権限あり**」と表示
  (出てなければ管理者として再起動)

### デモ用 EVTX を用意

- [ ] `workspace/uploads/` に **デモ用 EVTX** を 1〜2 個置いておく
  (推奨: 既に解析済みの `sideloading_wwlib_sysmon_7_1_11.evtx` または
   `meterpreter_migrate_to_explorer_sysmon_8.evtx` — 既に検知が出ているもの)

### ブラウザ準備

- [ ] **F11 で全画面化** (パワポと往復するなら不要)
- [ ] **ブックマークバー / 拡張機能アイコン** を隠して画面を綺麗にする
- [ ] **ズーム 110% 〜 125%** に上げて遠くの人にも見えるように

### バックアッププラン

- [ ] **失敗時用のスクリーンショット** を `docs/presentation/screenshots/` に保存
  (リハーサル時に撮影しておく)
- [ ] **スライド側にも実画面のスクショ** を 1〜2 枚入れておけば最悪安心

## ステップ 1: スキャンタブで EVTX 解析 (60 秒)

### 操作
1. **スキャンタブ** が表示されている状態にする
2. 「**EVTX をここにドロップ**」ゾーンに、デモ用 EVTX をドラッグ&ドロップ
3. 「**スキャン開始**」ボタンをクリック
4. 右側のライブフィードが流れ始めるのを 5-10 秒見せる

### 話すこと
> 「これがスキャンタブです。一画面で **対象選択 → 設定 → 実行** の 3 ステップになっています。
> 試しにこの EVTX をドロップして……(操作) ……スキャン開始します。
>
> 右側に **検知がリアルタイムで流れていきます**。これは SSE という仕組みで、
> サーバから検知が出るたびに押し出されてきます。」

### 強調ポイント
- 「**3 ステップ**」
- 「**リアルタイム**」
- 「**ドロップしてボタン押すだけ**」

### 失敗時
- ドロップが効かない → 「ファイルを選択」ボタンを使う
- スキャンが時間かかる → 「**既にスキャン済みのものを見せます**」と結果タブに切替

---

## ステップ 2: 結果タブで検知詳細 (60 秒)

### 操作
1. **結果タブ** に切り替え
2. ジョブ一覧から **「open」** で開く
3. 検知一覧から **critical / high の検知を 1 行クリック**
4. 下に解説パネルが展開、**サブタブ「概要」が表示**

### 話すこと
> 「結果タブに切り替えます。スキャン履歴があって、開くと検知一覧が出ます。
>
> 検知をクリックすると、こうやって **解説パネル** が下に展開します。
> ここに **'何を検知しているか'**、**'重要度の意味'**、**'ATT&CK 技術 (日本語)'**、
> **'次にすべきこと'** が全部出てきます。
>
> アナリストは普段、検知の意味調べるのに ATT&CK サイトに飛んだり Sigma の YAML
> 読んだりするんですが、それを **その場で完結** させてます。」

### 強調ポイント
- 「**ATT&CK が日本語**」
- 「**次にすべきこと**」 ← これが書いてある DFIR ツールは珍しい
- 「**その場で完結**」

### 失敗時
- 解説パネルが表示されない → ハードリロード (Ctrl+Shift+R)、または別の検知を選ぶ
- ルール解説が出ない → 「**ルール定義が見つからない場合は省略します**」と注釈

---

## ステップ 3: プロセスツリーサブタブ (30 秒)

### 操作
1. **同じ検知パネルの中で「プロセスツリー」サブタブをクリック**
2. ツリーが表示されたら **focal プロセス (オレンジ枠) を指差す**
3. 親を遡って 1-2 段見せる

### 話すこと
> 「同じパネルの中の **プロセスツリーサブタブ**。Sysmon EID 1 から
> **親子関係を再構築** しています。
>
> オレンジ枠が **検知のプロセス本体**。その親プロセスがここ、さらに親がここ……
> と遡れます。
>
> **'この malware の親は誰?'** が一目で分かる。」

### 強調ポイント
- 「**オレンジ枠が focal**」
- 「**遡れる**」

### 失敗時
- ツリーに何も出ない → 「**このホストには Sysmon EID 1 が無いケースです**」
  と説明、別の検知を選び直す or 飛ばす

---

## ステップ 4: ハントタブ (60 秒)

### 操作
1. **ハントタブ** に切り替え
2. プリセット「**痕跡隠蔽行為**」または「**直近24h重大**」をクリック
3. 結果が表示されたら **ピボット「ルール別」** をクリック
4. **「ホスト別」** にも切り替えて見せる

### 話すこと
> 「ハントタブ。**仮説駆動の検索** ができます。
>
> 例えば 'AAA' プリセットを押すと…… (クリック) ……該当する検知が
> 絞り込まれます。
>
> ここがいいのが **5 種類のピボット** がワンクリックで切り替わること。
> **ルール別** で見ると 'どのルールが多発しているか'。
> **ホスト別** で見ると 'どのホストが影響受けているか'。
>
> SOC で **'集計の view を変えたい'** という需要に応えてます。」

### 強調ポイント
- 「**仮説駆動**」
- 「**ワンクリック**」
- 「**集計の view を変える**」

### 失敗時
- プリセットで結果が 0 件 → 「**全期間で再検索します**」と時間範囲を変える

---

## ステップ 5: ダッシュボードの振舞い異常 (60 秒)

### 操作
1. **ダッシュボードタブ** に切り替え
2. 上の方の KPI / ドーナツ / 時系列を **3 秒だけ** 見せる
3. スクロールダウンして **「検知パターン異常」カード** を表示
4. **バーストの行を 1 つクリック** → ハントタブに遷移

### 話すこと
> 「ダッシュボード。KPI、重要度内訳、時系列が並んでます。これは普通の集計。
>
> 大事なのが下のここ、**'検知パターン異常'** カード。
> Sigma では書けない **'いつもと違う'** をここで自動抽出してます。
>
> 例えばこれ、**'Potentially Malicious PwSh' が平常の 2,492 倍** バーストしてます。
> 1 個 1 個の検知としては critical じゃないけど、**この量** は明らかにおかしい。
>
> 行をクリックすると…… (クリック) ……ハント検索に遷移して、該当する
> 検知をすぐ見られます。」

### 強調ポイント
- 「**Sigma では書けない'いつもと違う'**」
- 「**2,492 倍**」(ゆっくり言う)
- 「**自動抽出**」

### 失敗時
- バーストが見えない → 「**沈黙の例を見せます**」と別の種別をクリック
- 何も出ていない → 「**実データで動く例はこちらです**」とスライドの画像を見せる

## ステップ 6: ホストタブ (30 秒)

### 操作
1. **ホストタブ** に切り替え
2. **最上位 (LAPTOP など) の行を指差す**
3. 行をクリック → 詳細展開
4. **TOP ルール** をスクロールして見せる

### 話すこと
> 「最後にホストタブ。**リスクスコア順** でホストが並んでます。
>
> 一番上が最優先のホスト。**87.7 ポイント** と出てて、これは
> **検知数 + 重要度 + 直近度** を log 圧縮した値です。
>
> クリックすると詳細パネルが開いて、**このホスト固有の TOP ルール**、
> **検出チャネル**、**時系列** が見られます。
>
> **'どこから手をつける?'** が即答できます。」

### 強調ポイント
- 「**リスクスコア順**」
- 「**どこから手をつける?**」

### 失敗時
- ホスト 1 個しかない → 「**実フリートではこのように複数並びます**」とスライドの例を出す

## デモ全体での注意事項

### 🎯 やるべきこと

- **マウスポインタは大きめに**: Windows 設定でカーソルサイズを上げておく
- **クリック前に 'ここを押します'**: 動作の予告で見やすくなる
- **30 秒以内で次へ**: 1 画面に居座らない
- **画面を指差す**: 言葉だけでなく **どこを見て欲しいか** を示す
- **会場の後ろの人**を意識: 文字が読めるかどうか

### ❌ やってはいけないこと

- **ブラウザのアドレスバーを長時間映す**: ポート番号バレるしかっこ悪い
- **DevTools 開きっぱなし**: F12 が誤押下されてたら閉じる
- **コンソールエラーがあるまま見せる**: 事前にチェック
- **画面が固まったら焦る**: 「**少し待ってください**」と落ち着いて
- **マウスをぐるぐる動かす**: 緊張すると無意識にやりがち、ゆっくり

## 緊急時のエスケープ

### サーバが落ちた

1. デモを止める
2. 「**少々お待ちください**」
3. ターミナル開いて `.\start.ps1` 再実行
4. 立ち上がる間にスライドの **数字や Before/After** を補足解説
5. 立ち上がったら続行

### ブラウザが固まった

1. 「**ハードリロードしますね**」と Ctrl+Shift+R
2. それでも駄目なら **タブを閉じて開き直す**
3. それでも駄目なら **スクリーンショットで代用**

### Wi-Fi が落ちて IoC 更新失敗

1. デモ的には関係ない (ローカルツールなので) → 続行
2. 何か言われたら「**この動作は一度ローカルにキャッシュされているので OK です**」と説明
